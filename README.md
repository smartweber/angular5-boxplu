# MyApp

This project was generated with [Angular CLI](https://github.com/angular/angular-cli) version 1.1.3.

## Development server

Run `ng serve` for a dev server. Navigate to `http://localhost:4200/`. The app will automatically reload if you change any of the source files.

## Code scaffolding

Run `ng generate component component-name` to generate a new component. You can also use `ng generate directive|pipe|service|class|module`.

## Build

Run `ng build` to build the project. The build artifacts will be stored in the `dist/` directory. Use the `-prod` flag for a production build.

## Running unit tests

Run `ng test` to execute the unit tests via [Karma](https://karma-runner.github.io).

## Running end-to-end tests

Run `ng e2e` to execute the end-to-end tests via [Protractor](http://www.protractortest.org/).
Before running the tests make sure you are serving the app via `ng serve`.

## Further help

To get more help on the Angular CLI use `ng help` or go check out the [Angular CLI README](https://github.com/angular/angular-cli/blob/master/README.md).

##How to correctly compile SASS

Navigate to the trunk/src/app/styles folder. Run the following command:
sass --watch .:../.. --style compressed
It will compile directly to the styles.css file in the src folder

##VERY USEFUL - How to exclude SASS/CSS changes from reloading the page
Use the --hmr flag when running ng serve:
ng serve --hmr
You can now edit your SASS files, see the changes in real time AND not have the page completely reload
